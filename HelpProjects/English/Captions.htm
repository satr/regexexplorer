The description of operating modes - "Captions" (the tab "Matches/Captions")

"Captions" - sample of a collection of lines matched to a pattern

Allocation from the text (including multiline text) collections of the lines which mutch to a pattern of regular expression

The Operating procedure:

  1. Into a field "Target text" to enter the text which will be checked on conformity to a pattern
  2. Into a field "Regex pattern" to enter the text of a pattern
  3. To press button "Check" or a key "Enter" (if the cursor is in a field of input of a pattern)
  4. In a field "Result" will be displayed result of check: "There are Captions" or "No matches"
  5. In the list of "Captions" the collection of lines from the checked text, which mutch to a pattern of regular expression will be displayed

Sample:

In the pattern field enter:
(?<1>\w+)\s+(?<1>ca\w)

Choose the mode "Matches/Captions" (the tab with same name), inside choose the tab "Captions"

In the field "Target text" enter the text:
gray cat, yellow cab, green car

Click the button "Check"

In the list "Captions" the collection of captions will be displayed - a pair per line, placed in the same group:

Change the pattern:
(?<1>\w+)\s+(?<2>ca\w)

Click the button "Check"

In the list "Captions" the collection of captions will be displayed - a single value per line

placed in two groups

which can be selected from the drop down list

In the field "Captions result pattern" enter a pattern:
$1;$2;

Click the button "Check"

In the list "Captions" the collection of captions will be displayed, made according to the result pattern. In the drop down list, in addition to "Values by result pattern", groups still can be selected.

In "Captions result pattern" a number, followed by the symbol "$", defines a number of caption to be placed in this position. Rest symbols are any necessary.

Samples of "caption result patterns":
Color: $1. Object: $2.
$2 has a color $1.
This color is "$1",
