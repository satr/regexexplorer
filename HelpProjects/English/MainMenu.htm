"RegexOptions" (options of regular expression)

In section " Regex pattern " (the pattern of regular expression) displays/hides Subsection of options of regular expression:

  • "IgnoreCase"(to not distinguish the register) - identical symbols in different registers Are perceived as equivalent (for example: "R" it is equivalent "r")
  • "IgnorePatternWhitespace"(to pass blanks) - all blanks in a pattern Regular expression are ignored (for example: "IsMatch" it is equivalent "Is Match") and it is authorized Use comments (beginning a symbol "#")
  • "Singleline"(the One-line mode) - changes value of a symbol of a point ".", So it means "any symbol", instead of "any symbol excepting \n"
  • "Multiline"(the Multilower case mode) - changes value of symbols " " and "$" (Meaning has begun also the end) Is every substring mutch to a target line, instead of to all target line as a whole
  • "ExplicitCapture"(the Certain allocation) - will define, that only correct Elements are obviously called or numbered in groups of the form (?<name>Å). It allows not called To expressions to operate as independent groups without syntactic infringements
  • "RightToLeft"(From right to left) - search will be is made from right to left
  • "Compiled" (It is compiled) - regular expression is compiled in assembly. It increases productivity, but waste more time at start
  • "CultureInvariant"- features of regional settings
    • do not differ
  • "ECMAScript"- support of ECMAScript-compatible behaviour for regular Expressions. The option can be used only together with options: IgnoreCase, Multiline and Compiled
